# MEM_009 API 字段映射核对

## 铁律

**任何新 API 字段映射必须用 curl 验证数据正确性。** 东方财富 API 字段名是数字编号，肉眼极易抄错。字段编号有近似值（f47 vs f104），直觉不可信。

## v1.1.1.2 实踩案例

`fetchMarketBreadth()` 初始实现用 `f47,f48,f50,f51,f170`：

| 字段 | 认为的含义 | **实际含义** | 实际 API 返回值 |
|------|-----------|-------------|----------------|
| f47 | 上涨家数 | 每股收益（个股财务指标） | 705018739 |
| f48 | 下跌家数 | 每股未分配利润（个股财务指标） | 1447463947685.1 |

这些极端脏值被 `.sane()`（`0..10000` 边界）过滤为 0。然后 `if (up == 0 && down == 0)` 的早期 return 把有效的 `f50`（涨停家数）也一并丢弃，导致情绪始终为空。

## 正确字段

| 字段 | 含义 |
|------|------|
| f104 | 上涨家数 |
| f105 | 下跌家数 |
| f106 | 平盘家数 |
| f50 | 涨停家数 |
| f51 | 跌停家数 |

## curl 验证命令

```bash
curl -s 'https://push2.eastmoney.com/api/qt/stock/get?secid=1.000001&fields=f104,f105,f50,f51' \
  -H 'Referer: https://quote.eastmoney.com/' | python3 -m json.tool
```

## 防御措施

- `.sane()` 边界 0..10000 保底，但**不能替代字段正确性**——本例就是 `.sane()` 通过了（返回 0）但逻辑错误
- `toSentimentSlice()` 外层条件必须包含全部四个字段：`up>0||down>0||limitUp>0||limitDown>0`，不要因为部分为空就丢弃其他有效字段
- 情绪抓取日志必须有四种反馈：有效数据 / 快讯 / 暂无数据 / 获取失败
